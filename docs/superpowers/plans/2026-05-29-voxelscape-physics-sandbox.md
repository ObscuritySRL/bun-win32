# Voxelscape Physics Sandbox Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Turn the raytraced `voxelscape` fly-through into a fully playable voxel physics sandbox (sand/gravel/water/lava/fire cellular sim, swept-AABB player physics, chain-reaction TNT with bullet-time, AI critters, projectiles, debris, day-night, HUD, procedural audio) — built to be spectacular and shareable.

**Architecture:** One unified raytrace. Two `uint32` grids (`world` authoritative, `frame` = world + stamped dynamic voxels uploaded each render). A pure, testable `Sim` (grid + active-set cellular automata + explosions) is separated from the game layer (entities, FX, audio, render). Non-solid glow (explosions/fire/lava/fuses) rides an in-shader additive depth-occluded glow-point layer. Player and all entities share one swept-AABB integrator.

**Tech Stack:** Bun + TypeScript, Win32 FFI via `@bun-win32/*`, D3D11 through `_gpu.ts`, HLSL (runtime-compiled ps_5_0/vs_5_0), `_hud.ts` (GDI→backbuffer HUD), `_audio.ts` (XAudio2 PCM), `_snapshot.ts` (PNG capture).

**Verification model (read first — deviates from default pytest-TDD by repo constraint):**
- **Type gate (every task):** `cd packages/all && bunx tsc --noEmit` → 0 errors. (Confirm exact invocation in Task 1; repo pins `linker=hoisted` so transitive `@types` resolve.)
- **Logic TDD:** pure `Sim`/physics functions are exercised by `packages/all/example/voxelscape.logic.test.ts`, **run with `bun run` (NOT `bun test`, which segfaults repo-wide)**. It prints `PASS`/`FAIL` lines and exits non-zero on failure. This is the red/green loop for simulation rules.
- **Visual verification:** rendering/shader/HUD/feel are verified by running the demo and *looking* at frames:
  - Interactive: `bun run packages/all/example/voxelscape.ts`
  - Capture (PowerShell): `$env:DEMO_DURATION_MS='12000'; bun run packages/all/example/voxelscape.ts; Remove-Item Env:\DEMO_DURATION_MS` → writes `packages/all/screenshots/voxelscape.png`; open/Read the PNG and confirm it looks right (non-black, scene present). Per repo rule, never trust numeric-only checks for rendering.
- **Frequent commits:** one per task (after tsc + relevant logic/visual checks pass).

**Single-file note:** All game code stays in `packages/all/example/voxelscape.ts` (matching every other demo + the `voxelscape` npm script + screenshot pipeline). `main()` is guarded by `if (import.meta.main)` so the logic test can import pure exports without launching a window. The TS block palette is the single source of truth; the HLSL palette is generated by interpolating those constants into the shader string so they cannot drift.

## Locked shared interfaces (used by many tasks — keep names exact)

```ts
// Dimensions (existing): i = x + z*W + y*W*D
const W = 128, D = 128, H = 48, SEA_LEVEL = 16;

// Block ids (single source of truth; HLSL palette interpolates these).
const B_AIR=0, B_GRASS=1, B_DIRT=2, B_STONE=3, B_SAND=4, B_WATER=5,
      B_WOOD=6, B_LEAF=7, B_SNOW=8, B_GRAVEL=9, B_TNT=10, B_LAVA=11,
      B_GLOWSTONE=12, B_PLANK=13, B_BRICK=14, B_OBSIDIAN=15;
const B_COUNT = 16;

// Per-block property tables, indexed by block id (length B_COUNT).
const SOLID:        boolean[]; // collides with bodies (water/lava/air = false)
const FALLS:        boolean[]; // sand-like settling (SAND, GRAVEL)
const FLUID:        number[];  // 0=none, 1=water, 2=lava
const FLAMMABLE:    number[];  // 0=no, else ignite-chance weight (LEAF high)
const EMISSIVE:     number[];  // 0..1 self-emission (LAVA, GLOWSTONE)
const BLAST_RESIST: number[];  // 0..1 (OBSIDIAN=1 immune, BRICK high, soil low)
function isSolid(b:number):boolean; // SOLID[b]

// Pure simulation (factory → testable at any size).
interface ExplodeResult { destroyed: {x:number;y:number;z:number;block:number}[]; litTNT:number; }
interface SimEvents { explosions: {x:number;y:number;z:number;power:number;result:ExplodeResult}[];
                      steam:{x:number;y:number;z:number}[]; splashes:{x:number;y:number;z:number}[];
                      extinguished:{x:number;y:number;z:number}[]; }
interface Sim {
  W:number; H:number; D:number;
  world: Uint32Array;
  active: Set<number>;
  fuses: Map<number,number>;    // cellIndex -> seconds left
  burning: Map<number,number>;  // cellIndex -> seconds left
  idx(x:number,y:number,z:number):number;
  inBounds(x:number,y:number,z:number):boolean;
  getBlock(x:number,y:number,z:number):number;
  setBlock(x:number,y:number,z:number,t:number):void; // wakes cell + 6 neighbors
  wake(x:number,y:number,z:number):void;
  light(x:number,y:number,z:number):void;   // begin a TNT fuse (~1.5s + jitter)
  ignite(x:number,y:number,z:number):void;  // begin burning a flammable cell
  stepTick(budget:number):SimEvents;         // ONE 20Hz world tick, ≤budget cell visits
  explode(x:number,y:number,z:number,radius:number,power:number):ExplodeResult;
}
function createSim(W:number,H:number,D:number, data?:Uint32Array):Sim;

// Pure swept-AABB mover. sampleSolid(x,y,z) → does that voxel block a body?
interface MoveResult { x:number;y:number;z:number; vx:number;vy:number;vz:number;
                       onGround:boolean; hitX:boolean; hitY:boolean; hitZ:boolean; }
function sweptMove(px:number,py:number,pz:number, vx:number,vy:number,vz:number,
                   sx:number,sy:number,sz:number, dt:number,
                   sampleSolid:(x:number,y:number,z:number)=>boolean,
                   stepUp:number):MoveResult;
```

`createSim` owns `world`; the game owns `frame` (= copy of `world` + entity stamps). `explode` carves + wakes + lights chain TNT + returns destroyed cells; it does NOT spawn entities/FX (the game does, from the result). Fuse expiry inside `stepTick` calls `explode` and accumulates into `SimEvents.explosions`.

## Task 1.5: Visual overhaul — lighting, shadows, water (user-requested, done early so all later work sits in the final look)

**Files:** Modify `voxelscape.ts` (HLSL only). Visual verification (capture + inspect; compare before/after).

**Why early:** user feedback — block shadows look washed/muddy and the water reads as flat cyan. Fixing the look now means every subsequent screenshot is gorgeous and feel-judgements are made against the final art.

- [ ] **Step 1: Smooth (per-corner) ambient occlusion.** Replace the single-value `ambientOcc` with Minecraft-style smooth lighting: sample the 3 occluders (2 edges + 1 corner) at *each* of the face's 4 corners and bilinearly interpolate across the face using the in-face UV. Result is a smooth AO gradient that darkens crevices/edges crisply instead of a flat per-face tint.

- [ ] **Step 2: Crisp directional face shading + less wash.** Add fixed per-face weights (top brightest ~1.0, +X/−X ~0.80/0.66, +Z/−Z ~0.74/0.70, bottom ~0.5) multiplied into the lit color so cubes read as cubes. Reduce the sky-ambient magnitude and pull the distance-fog/haze start farther out + lower its strength so the foreground isn't pastel-washed; keep the golden-hour sun but raise lit-vs-shadow contrast.

- [ ] **Step 3: Crisper sun shadows.** Add a proper normal-offset bias to the shadow-ray origin, jitter the ray direction slightly (hash dither) for a soft-but-defined penumbra, and tune the visibility range so contact shadows under blocks/trees are clearly darker (not the current muddy 0.12–0.42 half-tone).

- [ ] **Step 4: Water — real reflections.** In the water branch, compute the reflected ray off the wobbled surface normal and `traceVoxels` along it (capped steps): if it hits terrain, shade that hit with cheap lighting (albedo × ndl × sky-ambient); else `skyColor(refl)`. So the water mirrors the actual hills/trees/sky.

- [ ] **Step 5: Water — depth color, transparency, foam.** Probe downward from the surface to find the floor depth; tint deep→shallow (deep teal `~(0.02,0.12,0.18)` → turquoise `~(0.10,0.42,0.46)`), blend in the *bottom block color* for shallow water (see-through refraction), Fresnel-mix refracted vs reflected, keep a sharp specular sun glint on ripple normals, and add a white **foam** band where the water is shallow or adjacent to a non-water block. Tune ripple normal frequency for crisp small waves.

- [ ] **Step 6: Verify (visual).** tsc 0; capture a frame (`$env:DEMO_DURATION_MS='7000'; bun run …voxelscape.ts; Remove-Item Env:\DEMO_DURATION_MS`) and Read the PNG. Confirm: blocks have crisp readable shadows/AO, foreground isn't washed out, and the water shows reflections + depth + foam (not flat cyan). Iterate until it's beautiful. Send before/after to the user.
