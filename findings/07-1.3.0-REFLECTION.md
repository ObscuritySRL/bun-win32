# bun-uia 1.3.0 — Recursive Reflection (2026-06-14)

> Produced by a 16-agent Opus/ultracode workflow: 6 reflection lenses (package limitations, MCP-surface
> gap, human-parity, competitive, Anthropic/MCP spec, perf/correctness) → dedupe/rank → 8 adversarial
> verifiers (5 ship / 3 defer / 0 reject) → synthesis. 77 raw findings. Then governed by the end-user's
> three reframes this session (below). Every load-bearing claim cites repo `file:line` or a primary URL;
> the verifiers' job was to REFUTE, and they caught several fabrication traps in the prior roadmap.

## End-user reframes (this session) — these reorder the value function

1. **I am the end user.** No planning handoff; I (the AI) pick and ship what *I* need to control Windows.
2. **Use every information-retrieval path** about windows/controls, not just actions.
3. **Transcend human limits.** A human needs a window foregrounded, visible, under a cursor, on the
   active desktop. I do not. **Cursor-free / no-foreground / offscreen is the DEFAULT, not the fallback.**
   Screenshots/SendInput are only for (a) a human watching or (b) pixel-only surfaces with no semantic layer.
4. **The deployer chooses comfort level.** The developer wiring bun-uia to their agent must be able to
   specify which capabilities/interaction-modes are enabled — first-class, safe-by-default, configurable.

## Fabrication traps the verifiers caught (do NOT ship these as "free wiring")

- **`locate_image` is NOT free.** `findImage`/`locateOnScreen` (match.ts) take a raw-RGB `Bitmap`; the
  package has **no image decoder** (png.ts is encode-only — grep confirms only `decodeBstr`/`decodeStyles`).
  A base64 PNG from an MCP client cannot be turned into RGB. → **Deferred** to a later increment gated on a
  real PNG decoder (inflate + de-filter), or an interim raw-RGB-needle tool. Shipping it as-is = fabrication.
- **"Background z-order raise works"** is false. `SetForegroundWindow` is denied from a background process;
  `SetWindowPos(HWND_TOP, SWP_NOACTIVATE)` is a *best-effort restack only*, not a true bring-to-front (MSDN).
  → We don't NEED it: cursor-free UIA `invoke` acts on occluded/minimized windows regardless of z-order.
- **`safeExecute` cannot wrap the MCP ref-dispatch.** It's a selector-batch executor; `redactTree` is
  `UiaNode`-typed and the RefNode snapshot renders no control value. → Author a **fresh** policy gate using
  only the `SafeOptions` *shape*, not `safeExecute`/`redactTree`.
- **Cached-state surfacing is segfault-gated.** Adding `IsOffscreen`/`HasKeyboardFocus`/`Value` to the
  snapshot needs `GetCachedPropertyValue` + a VARIANT reader that **does not exist** in reads.ts → new slot,
  segfault risk. → **Deferred** behind a live-proof spike. The pure-TS half (broaden the interactive set) ships.

## (a) Package limitations — ranked

1. MCP agent is BLIND beyond one window's PrintWindow grab (no 2nd monitor / composited / overlay / no-window). → `screen_capture` + `inspect_point`.
2. `screenshot` returns black on locked/DWM/Chromium/game surfaces with no fallback. → near-uniform check → `screenshotScreen({window bounds})`.
3. No drag / press-and-hold over MCP. → `drag`, `hold_key`.
4. No Set-of-Marks grounding image over MCP. → `screenshot_marked` (window-local marks; keep on window capture).
5. `locate_image` impossible today — no PNG decoder. → **DEFER**.
6. MCP never settles; re-sends whole tree each step. → `wait_idle` + diff-mode snapshots.
7. No window/monitor management; lone `SetForegroundWindow` is a denied no-op. → `manage_window` + `list_monitors`; restack documented as best-effort.
8. Native-Win32 / MSAA fallbacks unreachable from MCP. → `native_tree` + `msaa_tree`.
9. Snapshot omits control state; INTERACTIVE set misses Tab/DataItem/Header/Custom. → broaden set now (pure-TS); cached state DEFER.
10. `nameContains`/RegExp selector marshals every descendant. → DEFER to slot-24 live-proof.
11. No launch/run from MCP; safety gate unused. → fresh policy gate + gated `launch_app`/`run_program`/`open`/file tools.
12. `computer_20251124` drift: no zoom/region, no modifier-clicks. → add region+zoom, thread `action.text` modifiers (Stage, computer.ts).
13. `Element.parent` re-acquires a fresh ControlViewWalker each call (3 round-trips/level). → memoize walker, measure-gated.
14. Clipboard text-only. → CF_DIB image + CF_HDROP files.
15. No until-waits / waitForWindow. → generalized `waitFor` + `waitForWindow`/`waitForWindowClose` + typed UiaError.
16. **SLOT table has ZERO automated coverage; a transposed slot SEGFAULTS.** → `slot-gate.test.ts` parsing UIAutomationClient.h.

## (b) Ways an AI still cannot fully use Windows — ranked

1. Can't SEE past one window. → `screen_capture`.
2. Can't turn a pixel into a control. → `inspect_point`.
3. Can't drag / hold. → `drag` / `hold_key`.
4. Can't manage windows/monitors / reliably raise occluded. → `manage_window` + `list_monitors` (+ honest restack note).
5. Can't settle / notice changes (events are a dead-end). → `wait_idle` + diff snapshots (polling is the supported substitute).
6. Can't read control state at a glance. → broaden interactive set now; cached state behind a spike.
7. Can't start a program / open a URL; no trust gate. → gated launch/run/open + policy gate.
8. Can't reach web DOM / OS-events / cross UAC — hard walls. → honest docs: pixel layer for web, polling for events, detect-and-surrender at UAC. Never claim a bypass.
9. Can't copy/paste images or files. → CF_DIB / CF_HDROP.
10. Can't wait on real-app conditions. → until-waits + waitForWindow + UiaError taxonomy.

## Committed 1.3.0 — "Drive Windows in the dark"

Reordered around reframe #3: cursor-free / no-foreground / offscreen is the headline. Every shipped item is
pure-TS or a verified low-risk Win32 read — **zero new COM vtable slots, zero segfault / foreign-thread /
fabrication risk**. Each proven by driving a real app and LOOKING at the bytes.

- **Senses (read, in the dark):** `screen_capture`, `inspect_point`, `screenshot_marked`, `native_tree`,
  `msaa_tree`, enriched `list_windows` (exe path, min/max/foreground, monitor), `list_monitors`,
  `inspect_element` (PROVEN getters only), `wait_idle`, diff-mode snapshots, screenshot black-fallback.
- **Reach (act, in the dark):** `click` reworked cursor-free-first (drop the denied `SetForegroundWindow`;
  real cursor only on explicit opt-in), `drag`, `hold_key`, `manage_window` (move/resize/min/max/restore
  without foregrounding).
- **Beyond the GUI, gated default-OFF:** `launch_app` / `run_program` / `open` / file tools behind a fresh
  policy gate.
- **Policy/config layer (deployer's comfort level):** profiles `readonly` / `safe` (default) / `full`;
  `BUN_UIA_ALLOW`/`DENY`, `BUN_UIA_CURSOR`, `BUN_UIA_OS`, `BUN_UIA_FS_ROOT`, `BUN_UIA_CONFIRM`. `tools/list`
  advertises only enabled tools; blocked calls return a clear policy `isError`; actions audited to stderr.
  Confirm uses MCP elicitation when the client supports it (verify in spec), else deny-by-default.
- **Correctness:** `slot-gate.test.ts` diffs SLOT vs `UIAutomationClient.h`. Perf: memoize ControlViewWalker.
- **Deferred (no fabrication):** `locate_image` (PNG decoder), cached-state VARIANT reader, slot-24 substring,
  VirtualizedItem.Realize, Grid/Transform — all COM-slot / segfault-gated, for a later live-proof spike.

**Runner-up:** Observation & settle tools (wait_idle + diff + native/msaa trees) — folded INTO this increment
rather than shipped alone, since seeing+reaching in the dark is the wider unlock.
